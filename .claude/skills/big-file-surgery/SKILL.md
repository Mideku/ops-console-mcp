# Big-file surgery — search first, read narrow, never re-read what you just wrote

## Access (read/search) — pick by need

- **Find a specific symbol/key/string** (a tool name, a JSON property, a config key): use
  `Grep` with the exact string, `output_mode: "content"`, a small `-C` (0-3 lines of
  context), and a `head_limit`. This answers most questions without opening the file at all.
  Example: looking for where `bind_address` is validated — grep `collector/` for
  `bind_address`, don't open `collector.py` end to end.
- **Window around a known region**: check size first (`wc -c <file>` or `ls -la`), then
  `Read` with `offset`/`limit` around the line Grep pointed at, rather than the whole file.
  This matters most for the generated files under `src/**/obj/` and `tests/**/obj/`
  (`project.assets.json`, `*.nuget.g.props`) — these are build output, not source; you should
  rarely need to read them at all, and never edit them (they're regenerated by `dotnet
  restore`).
- **Tail-only files** (CI logs, long command output captured to a file): use `tail -n 60`
  via Bash, never a full `Read`.

## Edit

1. Make the smallest edit that fixes the target region — don't reformat or reflow
   surrounding lines you weren't asked to touch; a large diff on a shared file (README.md,
   ci.yml) makes review harder and increases the chance of an unrelated regression.
2. For structured files (JSON/YAML), validate after editing:
   `python -c "import json; json.load(open('collector/collector.config.example.json'))"`
   for JSON, or a YAML load for `.github/workflows/ci.yml`.
3. `git diff --stat` after the edit — confirm only the intended file(s) and region changed.

## Never

- Never `Read` a whole file you're about to Grep anyway — Grep first, decide if a window
  read is even necessary.
- Never re-`Read` a file immediately after you `Edit`ed it "to check the change landed" — the
  edit tool already confirms success; re-reading burns a round-trip for no new information.
- Never edit anything under `obj/` or `bin/` — it's build output, excluded by `.gitignore`,
  and regenerated on the next `dotnet restore`/`build`.
